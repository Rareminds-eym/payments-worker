# razorpay-api

Cloudflare Worker that acts as a shared Razorpay payment processing layer. It sits between your Pages Functions (business logic) and the Razorpay API — no frontend ever calls it directly.

```
Browser → Pages Functions (/api/payments/*) → razorpay-api Worker → Razorpay API
             signs JWT with RAZORPAY_SERVICE_SECRET ↗
```

---

## Prerequisites

- [Node.js](https://nodejs.org) v18+
- [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/) v3+
- A [Cloudflare account](https://dash.cloudflare.com/sign-up)
- A [Razorpay account](https://razorpay.com) with API keys

---

## Local Development

**1. Install dependencies**

```bash
npm install
```

**2. Create your local secrets file**

```bash
cp .dev.vars.example .dev.vars
```

Edit `.dev.vars` with your local credentials. This file is gitignored — never commit it.

**3. Create the KV namespace (one-time)**

```bash
npm run kv:create
```

Paste the returned `id` and `preview_id` values into `wrangler.toml` under the `[[kv_namespaces]]` binding.

**4. Start the dev server**

```bash
npm run dev
```

The worker runs on `http://localhost:9003` by default.

## Deployment

This is a single unified worker instance. `ENVIRONMENT` is a runtime var — not a separate worker per environment.

**1. Configure secrets (first time only)**

```bash
npx wrangler secret put RAZORPAY_SERVICE_SECRET
npx wrangler secret put RAZORPAY_KEY_ID
npx wrangler secret put RAZORPAY_KEY_SECRET
npx wrangler secret put RAZORPAY_WEBHOOK_SECRET
```

Verify:

```bash
npm run secrets:list
```

**2. Deploy**

```bash
npm run deploy                          # production
npm run deploy:development              # development env
npm run deploy:staging                  # staging env
```

## Authentication

All endpoints except `GET /health` require a signed service JWT in the `Authorization` header:

```
Authorization: Bearer <service-jwt>
```

The JWT is generated by the Pages Functions layer using the shared `RAZORPAY_SERVICE_SECRET`. The worker verifies the JWT and checks that `service_id === "functions-payment-service"`. No request from the browser ever reaches this worker directly.

---

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `RAZORPAY_SERVICE_SECRET` | Yes | Shared secret used to sign/verify service JWTs |
| `RAZORPAY_KEY_ID` | Yes | Razorpay Key ID (`rzp_test_*` or `rzp_live_*`) |
| `RAZORPAY_KEY_SECRET` | Yes | Razorpay Key Secret |
| `RAZORPAY_WEBHOOK_SECRET` | Yes* | Required when using `POST /verify-webhook` |
| `ENVIRONMENT` | Yes | Set in `wrangler.toml`; overridden via `.dev.vars` locally |
| `ALLOWED_ORIGINS` | No | Comma-separated CORS origin allowlist — set in `wrangler.toml [vars]`. Edit and redeploy to add/remove domains. Localhost origins auto-added for non-production. If unset, all CORS requests are blocked. |
| `RATE_LIMIT_KV` | No | KV namespace for distributed rate limiting; falls back to in-memory if unbound |

## API Endpoints

All endpoints except `/health` require `Authorization: Bearer <service-jwt>`.

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/health` | None | Basic health check |
| GET | `/health?deep=true` | Bearer JWT | Health check + Razorpay connectivity ping |
| POST | `/create-order` | Bearer JWT | Create a Razorpay order |
| POST | `/verify-payment` | Bearer JWT | Verify HMAC payment signature |
| POST | `/verify-webhook` | Bearer JWT | Verify Razorpay webhook signature |
| GET | `/payment/:id` | Bearer JWT | Fetch payment details |
| POST | `/subscription/:id/cancel` | Bearer JWT | Cancel a subscription |

### Example: Create Order

```bash
# Generate a service JWT first (done by Pages Functions in production)
curl -X POST http://localhost:9003/create-order \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <service-jwt>" \
  -d '{"amount": 99900, "currency": "INR", "receipt": "rcpt_001"}'
```

## Security Notes

- `.dev.vars` is gitignored — never commit secrets
- The frontend never calls this worker directly — all calls go through Pages Functions
- Service JWTs are short-lived and signed with `RAZORPAY_SERVICE_SECRET`
- Razorpay HMAC signatures are verified using constant-time comparison to prevent timing attacks
- `?deep=true` health check requires auth to prevent unauthenticated outbound Razorpay calls
- CORS origins are configured in `wrangler.toml [vars]` as `ALLOWED_ORIGINS` — edit and redeploy to add/remove domains, no source code change needed. Localhost origins are auto-added for non-production environments only.
- See [ARCHITECTURE.md](./ARCHITECTURE.md) for the full security model
